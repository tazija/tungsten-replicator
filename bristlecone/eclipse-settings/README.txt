This folder contains Eclipse settings for the Bristlecone project:

- codeformatter.3.1.xml:
    Code formatting rules (Eclipse 3.1) that should be applied to all files.
    A file should not be commited without first performing a Source->Format
    and Source->Organize Import.
    Load in Windows/Preferences/Java/Code Style/Formatter/Import

- codetemplates.3.1.xml: 
    Code templates to use with Bristlecone under Eclipse 3.1. You should tune the
    Comments/Type and Code/New Java File to set your own name in the template.
    Load in Windows/Preferences/Java/Code Style/Code Templates/Import
    
- checkstyle.config.4.0.xml:
    Checkstyle 4.0 plugin configuration (http://eclipse-cs.sourceforge.net/). 
    Go in Windows/Preferences/Checkstyle then choose New and in type select
    Project Relative Configuration. Then load the config file.
